# Recipes — Workflow Patterns for Common Scenarios

Each recipe shows the skill-chain for a specific job-to-be-done. They're narrative, not reference — read the whole thing before running, then execute.

| Recipe | Time | When to use |
|---|---|---|
| [Competitive landscape](competitive-landscape.md) | 2–4 hrs | Entering a market, producing a competitor memo, evaluating a pivot |
| [First discovery call](first-discovery-call.md) | ~2 hrs | You've booked your first customer/partner/target interview |
| [Weekly strategic review](weekly-strategic-review.md) | ~45 min | End of every work-week — the repeating cadence |
| [Pressure-test a pivot](pressure-test-a-pivot.md) | ~90 min | Before any hard-to-reverse strategic change |

## How to read a recipe

Every recipe follows the same shape:

1. **When to reach for this** — whether this is the right tool
2. **The loop** — the skill chain in one line
3. **Step-by-step** — what to do and why
4. **Post-recipe checklist** — what should be true after
5. **Common mistakes** — the failure modes

## If you're new here

Start with [first-discovery-call](first-discovery-call.md) if you have a call booked, or [competitive-landscape](competitive-landscape.md) if you don't. Both exercise the full memory/evidence/synthesis loop end-to-end, so you'll feel how the system works rather than just reading about it.

After your first recipe, adopt the [weekly-strategic-review](weekly-strategic-review.md) as your repeating Friday cadence. Reserve [pressure-test-a-pivot](pressure-test-a-pivot.md) for when the stakes warrant 90 minutes of slow thinking.

## Want to write your own recipe?

Recipes are just markdown. If you develop a workflow that works for your project (onboarding a new team member, processing a customer-advisory-board meeting, a quarterly board-prep cycle), drop it in this folder. One narrative doc per recipe; include the same sections as the ones above.
